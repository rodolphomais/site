Git

23/09/2016 11:02

O git é uma ferramente bastante útil para o controle de versão no desenvolvimento de um projeto, permitindo armazenar todas as versões do projeto ao longo do seu desenvolvimento e em caso de problemas com a versão atual é possível restaurar para qualquer versão anterior que esteja no repositório. O seu funcionamento ocorre de forma distribuída, ou seja, qualquer pessoa que realize uma cópia da versão atual, também copia todo o repositório, permitindo que qualquer máquina ajude o servido a se recuperar.
Os arquivos de um repositório possuem três estados:
1. Consolidado – Quando o arquivo está armazenado no repositório local (Estado após ser feito o “commit”)
2. Modificado – O arquivo foi modificado, mas, a mudança ainda não foi consolidada no repositório.
3. Preparado – O arquivo está selecionado para ser consolidado no próximo “commit”.

A imensa vantagem do Git é ser multiplataforma, ele pode ser usado em ambiente Windows, Linux, MAC OS X e Solaris, o download está disponível em git-scm.com/downloads

Controle de Mudanças: Bugzilla

22/09/2016 16:20

O Bugzilla é uma poderosa ferramenta que auxilia o controle de mudanças durante o desenvolvimento de um projeto. Ela permite que desde pequenos bugs até erros mais críticos possam ser reportados, assim como também pode ser usado para registro e controle de modificações que podem ser solicitadas usuários e aceitas pelo equipe de desenvolvimento para ser implementado no projeto.
O Bugzilla pode sser baixado neste link: www.bugzilla.org/download

Conceitos básicos do Controle de Versão

20/09/2016 09:30

Subversion

Um sistema de controle de versão centralizado permite a criação de um repositório para hospedar os arquivos do projeto de forma que permita que todos da equipe do projeto possam ter acesso aos seus arquivos para realizarem sua implementação e após implementado, possam armazenas suas modificações no repositório para que a nova versão esteja disponível para todos os membros da equipe.
Uma poderosa ferramente que permite esta função é o Subversion.
O download desta ferramente pode ser feito neste link: subversion.apache.org/download.cgi

Jenkins

19/09/2016 13:15

Em processos de desenvolvimento ágil, o uso de uma ferramenta de integração contínua é vital para o projeto. O Jenkins é uma ferramenta usada para o uso da integração contínua em um projeto.
Ela permite integrar e automatizar tarefas como compilação do projeto e testes. Essas tarefas se feitas manualmente, podem levar bastante tempo e afetaria gravemente o tempo para desenvolvimento do projeto. Caso ocorra algum erro em algum dos testes automatizados, a ferramenta pode notificar no momento que o problema ocorrer a todos os desenvolvedores do projeto. permitindo que a correção seja feita brevemente e que não acumule os erros que podem ocorrer ao longo do projeto. A extenção do seu arquivo é .war, permitindo que ele funcione em um servidor de aplicação como o Tomcat.
O Jenkins é disponibilizado no link jenkins.io/download
Além do link para download, também é disponibilizado um guia para instalação desta ferramenta.
